# k6 Commands Template — Team 07

This file is a command reference. It is not an executable script.

Official benchmark course:
CLOUD101

Seeded student file:
benchmark/students.csv

Important rules:
- The same k6 scripts must be used for Stage 1, Stage 2, and Stage 3.
- Only BASE_URL and output file name should change between stages.
- Official benchmark endpoint:
  GET /students/{student_id}/grades?course_code=CLOUD101
- Smoke tests must not finalize CLOUD101.
- Run all official benchmarks from the same client machine.
- Create the results folder before running k6:
  mkdir -p results

---

# 1) Official 30-minute benchmark commands

## Stage 1 — Single EC2

Replace <stage1-elastic-ip> with the real Elastic IP.

k6 run \
  --env BASE_URL=http://<stage1-elastic-ip> \
  --env COURSE_CODE=CLOUD101 \
  --env STUDENT_FILE=benchmark/students.csv \
  --summary-export results/stage1_official_summary.json \
  benchmark/official_read_benchmark.js

# 2) Live demo commands

The live demo script is:
benchmark/demo_scaleout.js

It should generate:
- 1800 requests/minute
- 6 minutes duration
- GET /students/{student_id}/grades?course_code=CLOUD101

---

## Stage 2 live scale-out demo

Purpose:
Show Stage 2 ASG scaling from 1 instance toward 2 and 3.

k6 run \
  --env BASE_URL=http://<stage2-alb-dns> \
  --env COURSE_CODE=CLOUD101 \
  --env STUDENT_FILE=benchmark/students.csv \
  --summary-export results/stage2_demo_summary.json \
  benchmark/demo_scaleout.js

Expected observation:
- ASG starts with desired capacity 1.
- Traffic goes through ALB.
- ALBRequestCountPerTarget exceeds target value 600.
- ASG scales out from 1 to 2 and toward 3.
- After load stops, scale-in should eventually become visible.

# 5) Notes

- Do not run official benchmark before seed data is ready.
- Do not run official benchmark before all three stages pass smoke tests.
- Do not manually stop resources during the 30-minute idle observation window.
- Official benchmark result must use http_req_duration avg from k6.
- Stage 2 must use:
  ASG min=1
  desired=1 at benchmark start
  max=4
  target tracking
  ALBRequestCountPerTarget
  target value=600
  warmup=60 seconds
